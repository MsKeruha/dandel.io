# Архитектура проекта dandel.io

## Обзор
**dandel.io** — это фулстек-платформа логистики и доставки грузов. Проект ориентирован на использование алгоритма багатокритеріального оцінювання (SAW) для вибору оптимального тарифу доставки за критеріями: ціна, час, безпека, викиди CO₂.

## Технологический стек
- **Frontend**: React 19, TypeScript, Vite. Маршрутизація здійснюється через React state або Router. Стили написані на Vanilla CSS (Premium UI: glassmorphism, micro-animations). Nginx використовується для розгортання статичних файлів у production-середовищі (Multi-stage Docker build).
- **Backend**: Python 3.11, FastAPI, Pydantic, SQLAlchemy.
- **База данных**: PostgreSQL (контейнеризована), міграції керовані через Alembic.
- **Фонові задачі**: Celery + Redis для розрахунків маршрутів, динамічної зміни статусів, симуляції переміщення транспортних засобів та призначення водіїв.
- **Моніторинг**: Логування реалізовано за допомогою `loguru` (консоль + файли з ротацією).
- **Безпека**: FastAPI захищено Rate Limiter-ом (`slowapi`) та налаштованими політиками CORS.

## Основні компоненти

### 1. Калькулятор SAW (Simple Additive Weighting)
Алгоритм використовується в процесі створення доставки (екран `CriteriaSelector`). Користувач задає вагу (відсоток важливості) для 4 критеріїв. Бекенд розраховує нормалізовані оцінки для кожного доступного тарифу (Економ, Експрес, Безпечний) і рекомендує оптимальний.

### 2. Гейміфікація та Бонуси
Клієнти отримують кешбек за кожну доставку у вигляді бонусів. Система має рівні лояльності ("Насіння", "Парашутик", "Суцвіття", "Золота кульбаба"). Бонуси можна використовувати для оплати наступних доставок.

### 3. Картографія та Маршрутизація
Використовується зовнішнє API OSRM (Open Source Routing Machine) для отримання реальних геокоординат та побудови маршрутів на мапі. `RouteMap.tsx` відповідає за відображення на фронтенді (Leaflet).

### 4. Ролі користувачів
- **Customer (Клієнт)**: оформлює доставки, відслідковує статусы, має доступ до чату підтримки та кабінету.
- **Driver (Водій)**: має окрему панель (Driver UI). Бачить призначені йому вантажі, може змінювати їх статус (в дорозі, доставлено) та завантажувати фото-підтвердження.
- **Admin (Адміністратор)**: керує автопарком, бачить глобальну статистику.

## Структура Docker
Проект розгортається через `docker-compose`, який піднімає 5 контейнерів:
1. `db`: PostgreSQL.
2. `redis`: Broker для Celery.
3. `celery_worker`: Воркер для фонових задач.
4. `celery_beat`: Планувальник для періодичних задач (напр. зміна статусів авто кожну хвилину).
5. `backend`: FastAPI сервер (порт 8000).
6. `frontend`: Nginx сервер зі зібраним React-додатком (порт 80).
