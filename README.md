# Dandel.io

Dandel.io — это современный фулстек-веб-сервис для логистики и организации грузоперевозок. Проект разрабатывался с акцентом на алгоритмический подбор оптимального маршрута и тарифа с использованием метода многокритериальной оценки (SAW).

## Описание логики и алгоритма

Ключевая особенность платформы — это модуль калькуляции доставки, который работает на базе алгоритма Simple Additive Weighting (SAW). Приложение не просто рассчитывает стоимость по километражу, а учитывает одновременно четыре критерия:

1. Цена (минимизация затрат).
2. Время (скорость доставки).
3. Безопасность (проезд через безопасные зоны, возможность сопровождения груза).
4. Экологичность (минимизация выбросов углекислого газа - CO2).

Клиент при оформлении заказа настраивает ползунками приоритетность каждого из критериев. Алгоритм (реализованный на бэкенде) нормализует данные, применяет весовые коэффициенты пользователя к базовым значениям, рассчитанным через картографический сервис OSRM, и выдает итоговый скоринг для трех тарифов: Эконом, Экспресс и Безопасный. Система автоматически рекомендует тот тариф, который набрал наибольший балл.

## Архитектура и основные компоненты

Проект разделен на несколько изолированных модулей (кабинетов) в зависимости от роли пользователя.

1. Кабинет Клиента:
   - Калькулятор доставки с динамическими ползунками.
   - Оформление заказа (с симуляцией платежного шлюза).
   - Мониторинг статуса активных доставок.
   - Настройки профиля и система лояльности (накопление бонусов).
   - Встроенный чат с техподдержкой.

2. Кабинет Водителя:
   - Дашборд с активными назначениями на текущего водителя.
   - Функционал изменения статусов заказа (Принято, В пути, Доставлено).
   - Механизм загрузки фото-подтверждений успешной доставки груза.

3. Панель Администратора:
   - Общая статистика по платформе (количество заказов, выручка, активный транспорт).
   - Управление автопарком (CRUD операции с автомобилями).
   - Глобальный мониторинг всех заказов в системе.

4. Фоновые процессы (Celery Worker / Beat):
   - Система использует фоновые задачи для освобождения транспорта после доставки.
   - Периодические задачи обрабатывают обновление статусов и логику, не блокируя основной API-сервер.

## Стек технологий

Frontend:
- Библиотека: React 19
- Язык: TypeScript
- Сборщик: Vite
- Стилизация: Vanilla CSS (без использования сторонних UI-библиотек для полного контроля над дизайном, включая эффекты glassmorphism и микроанимации).

Backend:
- Фреймворк: FastAPI (высокопроизводительный асинхронный фреймворк)
- Язык: Python 3.11
- Валидация данных: Pydantic
- Работа с БД: SQLAlchemy (ORM)
- Миграции: Alembic
- Аутентификация: JWT токены (OAuth2 Password Bearer)
- Картография: Интеграция с OSRM API для маршрутизации

База данных и инфраструктура:
- Основная СУБД: PostgreSQL
- Брокер сообщений: Redis (используется для Celery)
- Очереди задач: Celery
- Контейнеризация: Docker и Docker Compose (проект состоит из 5 контейнеров: frontend, backend, postgres, redis, celery_worker, celery_beat)
- Веб-сервер: Nginx (используется внутри frontend контейнера для раздачи статики и проксирования API-запросов)

## Запуск проекта

Проект полностью контейнеризирован и готов к запуску локально.
Для запуска достаточно выполнить команду в корневой директории проекта:

docker-compose up --build -d

После успешного старта контейнеров:
- Клиентская часть будет доступна по адресу: http://localhost:3000
- Документация API (Swagger UI) доступна по адресу: http://localhost:8000/docs
