# Handoff for Next AI Agent

Добро пожаловать! Если ты читаешь этот файл, значит ты берешься за продолжение разработки проекта **dandel.io**. Пожалуйста, внимательно изучи этот документ перед началом работы, чтобы понимать контекст, стек и строгие правила пользователя.

## 1. О проекте (About the Project)
**dandel.io (Розумна логістика кульбаби)** — это современная фулстек-платформа логистики и доставки грузов. 
Особенности проекта:
- **Мультикритериальный расчет (SAW алгоритм)**: система предлагает 3 тарифа (Эконом, Экспресс, Безопасный), анализируя вес, цену, время, экологию и риски.
- **Экологичность**: расчет выбросов CO₂, назначение электротраков, отображение сэкономленного CO₂.
- **Геймификация**: программа лояльности, накопление бонусов, которые можно тратить на оплату.
- **Премиальный дизайн**: glassmorphism, яркие зеленые/золотые акценты, микроанимации.

## 2. Стек технологий
- **Frontend**: React, TypeScript, Vite. **Стилизация**: строго Vanilla CSS. Использование TailwindCSS запрещено без явного одобрения пользователя. Дизайн должен быть на высшем уровне (Premium UI/UX).
- **Backend**: FastAPI, Python 3.11, Pydantic.
- **База Данных**: PostgreSQL (в Docker-контейнере), SQLAlchemy (ORM).
- **Миграции**: Alembic.

## 3. Критические правила от пользователя (ОБЯЗАТЕЛЬНО К ПРОЧТЕНИЮ)
- 🇷🇺 **Общение**: Отвечать пользователю **строго на русском языке**.
- 🇺🇦 **Интерфейс**: Тексты в самом проекте, статусы, заглушки, модалки — **на украинском языке**.
- ⚠️ **Миграции БД (Alembic)**: **Ни в коем случае НЕ генерируй миграции базы данных вручную и не проси систему их запустить (`alembic revision --autogenerate`).** Пользователь строго делает это сам. Можешь только обновлять/создавать модели SQLAlchemy в Python.
- 🚫 **Терминал**: Не пытайся бездумно запускать команды в терминале. Выполняй команды только если это критически важно и только через инструмент `run_command` (пользователь все равно должен их одобрить).
- 📝 **Планирование**: Создавай план (Implementation Plan) **только для сложных задач**. Для простых фиксов или верстки делай всё сразу.
- ❓ **Уточнения**: Если инструкции непонятны — переспроси. Не додумывай критические вещи.
- ⚖️ **Вариативность**: Если возникают спорные вопросы по архитектуре или логике — предлагай несколько вариантов решения на выбор.
- ✨ **Дизайн**: Приложение должно выглядеть "дорого" и вызывать WOW-эффект. Никаких базовых HTML форм. Используй тени, плавные переходы (`transition`), скругления и гармоничную палитру.

## 4. Что нужно реализовать в следующей сессии
Пользователь утвердил следующий список фичей (аудит). Это твоя главная задача:

1. **Кабинет Водителя (Driver UI)**
   - Роль `driver` уже существует в базе, но интерфейса для них нет.
   - Нужно сделать отдельный вид (dashboard) для водителей.
   - Водитель должен видеть назначенные ему доставки.
   - Возможность менять статус доставки ("Груз забрал", "В пути", "Доставлено").
   - Интерфейс для прикрепления "фото-доказательства" завершения доставки.

2. **Фоновые задачи (Background Workers - Celery + Redis)**
   - Сейчас статусы доставок меняются динамически при GET-запросах (`update_dynamic_statuses` в `deliveries.py`).
   - Нужно интегрировать Celery (с брокером Redis).
   - Вынести пересчет динамических статусов и освобождение транспорта в периодическую фоновую задачу (например, каждую минуту).

3. **Реальная картография (OSRM или Google Maps API)**
   - Текущий расчет дистанции использует прямые линии (формула Гаверсинуса по координатам).
   - Нужно заменить это на реальный Routing API (бесплатный OSRM или Google Maps, если пользователь даст ключ).
   - Высчитывать реальное время в пути и километраж по дорогам при создании маршрута.

4. **Симуляция оплаты (Payment Gateway Simulation)**
   - Перед финальным оформлением доставки добавить шаг оплаты.
   - Реализовать красивую форму ввода данных банковской карты (с валидацией длины, CVC).
   - Сделать симуляцию процесса транзакции (скелетоны/спиннеры, имитация "Связь с банком...", "Оплата прошла успешно"). Без реальной интеграции Stripe/LiqPay.

5. **Настройки профиля (Profile Settings)**
   - Доработать существующую вкладку профиля.
   - Добавить возможность изменять: Имя, Номер телефона, Пароль.
   - Добавить UI для смены аватарки.
